ü™ú STEP 1 ‚Äî Create a GitHub Repository

Go to GitHub and create a new repository.

Example name:

my-notes

Recommended settings:

  • Public repository

  • Can initialize with README (optional)

This repo will later host your website.

ü™ú STEP 2 ‚Äî Upload Your Obsidian Vault to GitHub

Open Git Bash inside your vault folder and run:

git init
git remote add origin https://github.com/username/my-notes.git
git add .
git commit -m "first upload"
git push -u origin main

What this does:

  • Tracks your notes

  • Uploads them to GitHub

  • Creates version history

At this stage:
Your notes exist online ‚Äî but not yet as a website.

ü™ú STEP 3 ‚Äî Turn the Repo Into a Website (Basic Test)

Go to:

Repo ‚Üí Settings ‚Üí Pages

Set:

Source: Deploy from branch
Branch: main
Folder: /root

GitHub will give a link like:

https://username.github.io/my-notes/

This confirms:

  • GitHub Pages is working

  • Hosting is active

BUT:
It will only show files ‚Äî not a real website.

That‚Äôs where Quartz comes in.

ü™ú STEP 4 ‚Äî Install Quartz (Website Engine)

Quartz converts markdown notes into a real website with:

  • Navigation

  • Dark mode

  • Backlinks

  • Graph

  • Search

  • SEO structure

First Attempt (Common Beginner Error)

Tried installing using:

git clone https://github.com/jackyzha0/quartz .

Error received:

fatal: destination path '.' already exists

Why?
Because the vault folder already had files inside it.

Fix

Instead:

  1. Download Quartz ZIP

  2. Extract it

  3. Copy files manually into the project folder

ü™ú STEP 5 ‚Äî Another Mistake: Wrong Folder Placement

Quartz got extracted as:

Obsidian_Vault/quartz-4/

This is incorrect.

Correct structure should be:

Obsidian_Vault/
  notes...
  quartz/
  package.json
  quartz.config.ts

Fix:
Move all Quartz files to the root.

ü™ú STEP 6 ‚Äî Install Node.js (Required)

Quartz requires Node.js.

Install from official site.

Then verify:

node -v
npm -v

If both show version numbers ‚Üí ready.

ü™ú STEP 7 ‚Äî Install Dependencies

Inside the Quartz folder:

npm install
npx quartz build

This creates:

public/

Important concept:
public = the generated website

ü™ú STEP 8 ‚Äî Major Structural Decision (Very Important)

At first, Quartz was inside the main vault.

Problem:

  • Obsidian started showing system files

  • Vault looked cluttered

  • Hard to manage

Better Architecture (Recommended)

Obsidian_Vault/   ‚Üí private notes
quartz-site/      ‚Üí website engine

This keeps:

  • Writing clean

  • Publishing controlled

  • Structure professional

ü™ú STEP 9 ‚Äî Understanding Quartz Content Logic

Quartz publishes ONLY what is inside:

content/

Meaning:
You control what goes public.

Smart strategy:

Private vault ‚Üí everything
content/ ‚Üí selected polished notes

ü™ú STEP 10 ‚Äî First Build Error

Mistake:
Ran command inside wrong directory:

/quartz-site/content

Error:

ENOENT: package.json not found

Fix:
Move one level up:

cd ..
npx quartz build

Lesson:
Always run Quartz commands where package.json exists.

ü™ú STEP 11 ‚Äî Preview Locally

Quartz v4 command:

npx quartz build --serve

Then open:

http://localhost:8080

At first ‚Üí 404 page.

ü™ú STEP 12 ‚Äî Critical Rule: Homepage Must Be index.md

Quartz requires:

content/index.md

Without it:
‚Üí Site shows 404

Fix:
Create:

index.md

ü™ú STEP 13 ‚Äî Windows Extension Trap

File looked like:

index.md

But actually:

index.md.txt

Windows hides extensions.

Fix:
Enable ‚ÄúShow file extensions‚Äù
Rename properly.

ü™ú STEP 14 ‚Äî Custom Page Title

Add frontmatter at top:

---
title: My Knowledge Hub
---

This replaces the default ‚Äúindex‚Äù title.

ü™ú STEP 15 ‚Äî Push Website to GitHub

Inside quartz-site:

git init
git add .
git commit -m "Initial website"
git branch -M main
git remote add origin https://github.com/username/my-notes.git
git push -u origin main

ü™ú STEP 16 ‚Äî Merge Conflict (Real Situation)

Error:

CONFLICT (add/add): README.md

Cause:
Both local and GitHub had a README.

Fix:

git add .
git commit -m "merge quartz site"
git push

ü™ú STEP 17 ‚Äî Website Not Loading (Final Configuration Issue)

Problem:
GitHub Pages was reading:

/root

But Quartz builds into:

/public

Fix:

Settings ‚Üí Pages
Branch: main
Folder: /public

If /public not visible:

git add -f public
git commit -m "add public build"
git push

Fully functional with:
